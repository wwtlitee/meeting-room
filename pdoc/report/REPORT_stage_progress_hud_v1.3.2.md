# REPORT_stage_progress_hud_v1.3.2

## 基本信息

- 日期：2026-05-28
- 版本：v1.3.2
- 负责人：Solazhu
- 范围：专家团 viewer 会场左上角阶段进度 HUD

## 修改内容

- 在整个会场画面的左上空白区新增 `stage-progress-hud`，不放入会议大屏内部。
- HUD 显示当前轮次/阶段与状态，例如 `第一轮 · 发言中`、`第一轮 · 投票中`、`结论 · 收束中`。
- 陪审团会议读取当前可见 `voteRound.activationIndex`，仅在投票轮刚激活的发言节点显示 `投票中`，后续发言恢复 `发言中`。
- 普通会议没有 `voteRound` 时自动使用当前 `turn.phase`，显示 `发言中` 或结论阶段的 `收束中`，避免陪审团专用逻辑污染普通会议流程。

## 影响范围

- 影响 React viewer 的舞台层 UI，不改变 session 数据结构。
- 普通会议、陪审团会议共用同一个 HUD 组件。
- HUD 位于 `.stage-canvas` 内，随会场整体缩放。

## 复核结果

- `npm run build` 通过。
- 当前陪审团 session 已在 Codex 内置浏览器验证：HUD 出现在会场左上空白区，显示 `初始 · 投票中`。
- 普通会议适配已通过代码路径复核：无投票轮时不会显示 `投票中`。
