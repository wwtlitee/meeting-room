# REPORT_seat_vote_buttons_settings_v1.4.0

## 基本信息

- 日期：2026-05-28
- 版本：v1.4.0
- 负责人：Solazhu
- 范围：专家团 viewer 席位前立体 A/B 投票按钮、设置面板、布局校准扩展

## 修改内容

- 新增 `SeatVoteButtons`，在每个非主持席位前渲染两枚圆形 A/B 按钮。
- 按钮使用 CSS 渐变、高光、暗边、底部阴影和按压动画实现立体凸起与亮灯效果，不新增图片资产。
- 陪审团会议自动读取现有 `voteSides` / `voteAnimations`，普通会议默认隐藏投票按钮。
- 移除铭牌上的旧 A/B 小标记渲染，避免状态贴片继续影响观感。
- 顶部导航新增常驻 `设置` 入口，包含：
  - `测试 A` / `测试 B` / `关闭测试`，普通会议也可临时预览按钮按下亮灯效果；
  - 布局校准开关；
  - 大屏幕、铭牌、投票按钮三类目标选择；
  - 复制 JSON、重置当前、清空全部。
- 布局数据新增 `voteButtons`，并写入 `window.__agencyVoteButtonPlacements`，方便后续固化按钮坐标。

## 影响范围

- 影响 React viewer 舞台层和设置面板，不改变会议 session schema 必填字段。
- 普通会议不会显示投票按钮，除非用户在设置中启用 A/B 测试预览。
- 陪审团会议继续使用原有投票数据源，只替换视觉表达。

## 风险与防护

- 第一版按钮默认锚点可能需要用户微调，因此已把投票按钮加入布局校准目标。
- 按钮当前位于舞台层上方，可能在个别座位与桌面或气泡发生局部重叠，需通过设置面板微调。
- 测试 A/B 只影响本地 viewer 状态，不写回会议 JSON，避免污染会议记录。

## 复核结果

- `npm run build` 通过。
- 当前普通会议默认投票按钮数量为 0。
- 设置中测试 A 后显示 6 组投票按钮，A 亮 6 个、B 亮 0 个。
- 设置中测试 B 后显示 6 组投票按钮，A 亮 0 个、B 亮 6 个。
- 关闭测试后按钮数量恢复为 0；旧铭牌 A/B 标记数量为 0。
