# REPORT_normal_meeting_vote_buttons_hidden_v1.4.10

## 变更时间

2026-05-28

## 背景

普通会议中开启布局拖动后，投票按钮与投票手臂会被全局渲染，导致非陪审团会议画面出现不该存在的 A/B 投票控件。

## 变更内容

- 将会议画面内投票控件渲染条件收窄为：真实投票流程、伸手测试、或布局校准选中投票相关目标时显示。
- 普通会议默认隐藏投票按钮；设置面板中选中投票按钮、肩膀锚点、初始手位、投票手位、手臂长度时仍可显示并调整。
- 更新设置面板提示文案，避免误导为普通会议会常驻显示投票按钮。

## 影响范围

- 影响文件：`assets/expert-meeting-viewer/react-viewer/src/App.jsx`
- 陪审团会议投票流程不受影响。
- 普通会议的铭牌、座位、大屏布局校准不再附带显示投票控件。

## 复核

- 已执行 `npm run build`，构建通过。
- 逻辑复核重点：`showVoteButtons` 仍覆盖真实投票与伸手测试；`layoutCalibration.enabled` 仅在投票相关校准目标下触发按钮渲染。
