# REPORT_role_position_names_v1.23.0

## 背景

260 个导入人格的中文展示名此前主要按“短、稳、好认”压缩，但部分名称更像任务标签或能力标签，例如“浏览自动”“反馈综合”“ZK管家”，在会议铭牌和发言记录里不像真实职位。

## 修复

- 全量更新 `references/ROLE_NAME_ZH.md` 第三列中文展示名，保留 `分组 + 原名` 查找键不变。
- 命名统一改为短职位口径，优先使用“设计师 / 工程师 / 分析师 / 策略师 / 顾问 / 专员 / 运营师 / 导演 / 架构师 / 维护员”等岗位后缀。
- 保持现有大屏安全约束：260 行不增减、展示名不为空、展示宽度不超过 6 个单位。
- 同步刷新当前 viewer runtime 快照里的正式参会角色显示名，避免当前页面和新映射口径不一致。

## 复核

- `scripts/test_meeting_trigger_contract.ps1` 已通过。
- `ROLE_NAME_ZH.md`：260 行，空值 0，超过 6 宽度单位 0。
