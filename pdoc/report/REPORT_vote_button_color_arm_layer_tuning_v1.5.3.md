# REPORT_vote_button_color_arm_layer_tuning_v1.5.3

## 背景

用户在陪审团会议画面中发现三处问题：A 暗绿不够暗，B 亮红不够亮；投票手臂抬起后偏向下方按钮，没有到两个按钮中间；按钮图层压在手臂上方，按下关系不符合预期。

## 修改内容

- 调整卡通圆柱投票按钮颜色：
  - A 暗态从偏亮绿收敛为更深的墨绿。
  - B 亮态改为更纯、更亮的红色，提高和暗态/绿色并列时的辨识度。
- 新增 `VOTE_ARM_TARGET_LIFT_Y = 8`，统一把投票手臂目标点上提 8px，让手臂更接近两个按钮中间。
- 将 `.seat-vote-buttons` 层级从 `z-index: 76` 调整为 `z-index: 60`，保持按钮在桌面上方，同时低于 `stage-vote-arm` 的 `z-index: 62`。

## 影响范围

- 仅影响投票按钮颜色、手臂触碰目标和按钮/手臂图层关系。
- 不改变按钮坐标、旋转、大小、会议投票数据或手臂收回时序。

## 复核

- 已执行 `npm run build`，Vite 构建通过。
- 需要人工在当前 viewer 中确认：B 亮红是否足够明显、手臂是否抬到两按钮中间、手臂是否压在按钮上层。
