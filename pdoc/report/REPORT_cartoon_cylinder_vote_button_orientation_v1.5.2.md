# REPORT_cartoon_cylinder_vote_button_orientation_v1.5.2

## 背景

投票按钮已改为低矮卡通圆柱后，按钮组仍沿用用户校准过的旋转角度贴合桌边，导致圆柱本体也一起转向，出现椭圆面竖着、侧壁朝左右的问题。

## 修改内容

- 保留 `seat-vote-buttons` 的整体旋转、坐标和缩放，避免破坏用户已校准布局。
- 在每个投票按钮内部新增 `seat-vote-button-shape` 形状层。
- 通过 `--vote-button-upright-rotate` 抵消父级按钮组旋转，使每个圆柱按钮本体始终保持画面朝上、底边缘朝下。
- 按压动画仍挂在外层按钮上，继续沿既有方向下沉，不影响投票流程和手臂触碰时序。

## 影响范围

- 仅影响投票按钮视觉朝向。
- 不改变按钮坐标、旋转配置、投票状态机、手臂动画或会议 session 数据。

## 复核

- 已执行 `npm run build`，Vite 构建通过。
- 视觉效果需在当前内置浏览器中人工确认：重点看左右两侧旋转按钮是否都保持椭圆面朝上、底边缘朝下。
