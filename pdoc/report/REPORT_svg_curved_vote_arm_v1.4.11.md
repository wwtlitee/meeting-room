# REPORT_svg_curved_vote_arm_v1.4.11

## 变更时间

2026-05-28

## 背景

投票手臂在剪影方案下仍显得像直长柱，缺少自然弯曲和手端轮廓，和全黑牛牛剪影及卡通会议桌风格不一致。

## 变更内容

- 将投票手臂视觉层从 `span` 长条改为 SVG 闭合曲线剪影。
- SVG 手臂按现有 `length` 动态生成，保留肩膀锚点、初始手位、投票手位和手臂长度校准数据。
- 手臂轮廓采用肩部略宽、腕部略收、末端圆掌的形状，避免尖头或等宽长柱。
- 删除旧的 `stage-vote-arm-body` 与伪元素圆头，避免新旧形状叠加。
- 按钮触碰光点改为锚在 `--vote-arm-length` 位置，保持手端抵达按钮后再反馈。

## 影响范围

- 影响文件：
  - `assets/expert-meeting-viewer/react-viewer/src/App.jsx`
  - `assets/expert-meeting-viewer/react-viewer/src/styles.css`
- 不改变布局存储结构，不重置用户已固化的座位、铭牌、按钮、手臂坐标。
- 普通会议投票按钮隐藏逻辑继续保留。

## 复核

- 已执行 `npm run build`，构建通过。
- 逻辑复核重点：SVG 只替换视觉形状；投票流程、伸手测试、校准入口仍使用原状态与坐标。
