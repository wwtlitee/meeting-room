# PERSONA_STORE_SCHEMA

## 目标

`agency-agents` 的角色卡负责“怎么扮演专家”，人格资料层负责“这个专家长期学到了什么、近期记住了什么”。两者分离，避免把持续学习内容写回庞大的角色卡，也避免单个全局记忆文件无限膨胀。

## 目录结构

```text
references/personas/
├── <division>/
│   └── <role-slug>/
│       ├── profile.md
│       ├── knowledge.md
│       ├── memory.md
│       └── memory_summary.md
└── README.md
```

## 四类文件职责

| 文件 | 职责 | 增长策略 |
| :--- | :--- | :--- |
| `profile.md` | 稳定身份、职责边界、源角色卡入口 | 低频更新，只记录稳定人格事实 |
| `knowledge.md` | 专业学习资料、框架、术语、案例索引 | 原子条目，要求来源/适用范围，不写项目流水账 |
| `memory.md` | 近期任务记忆、偏好、上下文线索 | 只保留最近若干条原始记录 |
| `memory_summary.md` | 已压缩的长期记忆摘要与压缩策略 | 固定容量摘要，覆盖式更新 |

## 加载顺序

1. 选中角色后，先读取 `references/roles/<division>/<slug>.md`。
2. 再读取该角色的 `profile.md` 与 `memory_summary.md`。
3. 只有任务需要专业资料时，读取 `knowledge.md`。
4. 只有用户要求追溯近期细节，或当前任务明显依赖最近上下文时，读取 `memory.md`。

## 写入规则

- 不把长期学习资料追加到源角色卡。
- 不把所有人格写入同一个总记忆文件。
- `knowledge.md` 只放长期有效的专业资料；项目过程、一次性状态、临时 TODO 不进入知识库。
- `memory.md` 每条记录应能回答“做了什么、影响范围、为什么下次还会用到”。
- 禁止记录密钥、隐私、账号凭据、未授权个人信息。
- 不能把推测写成事实；不确定内容必须标注“待验证”。

## 压缩策略

- 默认 `memory.md` 保留最近 12 条原始记录。
- 旧记录压缩为 `memory_summary.md` 中最多 24 条摘要要点。
- 摘要优先保留稳定偏好、反复出现的判断准则、跨任务可复用的经验。
- 已被摘要吸收的旧原文可删除；如需审计，应在项目报告中另行保存证据，而不是让人格记忆无限增长。
- `scripts/compact_persona_memory.ps1` 提供机械压缩与容量控制；语义质量要求高时，由 Codex 先人工复核摘要再应用。
